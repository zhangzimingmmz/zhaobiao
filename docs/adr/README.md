# ADR（Architecture Decision Records）

> **作用**：记录关键架构/设计决策及选型理由。  
> **给谁看**：架构师、主程、资深开发、后续接手人。  
> **什么时候改**：重大决策时新增一条 ADR；旧决策被替代时新增新 ADR，一般不直接覆盖旧文档。  
> **谁改**：架构负责人、主程、发起重大改动的开发者。  
> **是否 review**：需要，且级别较高。

本目录存放单篇 ADR 文档，建议命名：`NNNN-简短标题.md`（如 `0001-采用SQLite作为主库.md`）。每篇应包含：背景、决策、后果/约束；可选替代方案与取舍理由。当前目录暂无 ADR 文件，后续按需新增。

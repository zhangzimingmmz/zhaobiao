# 01-项目导览

> **作用**：帮助第一次接触项目的人快速建立全局认识。  
> **给谁看**：新人、接手人、产品/管理者、AI 工具。  
> **什么时候改**：新增/下线功能、启动方式变化、功能边界或外部依赖变化时。  
> **谁改**：默认由改动功能的开发者先改。  
> **是否 review**：需要；功能边界变化大时建议模块负责人一起看。

---

## 1. 项目一句话简介

招投标信息平台：采集四川省公共资源交易平台与四川政府采购网等外部公告，经统一后端提供给微信小程序与运营后台；支持用户浏览、收藏，以及企业认证与运营侧审核、采集控制。

- **系统目标**：聚合招投标/采购公告，提供 C 端展示与 B 端运营能力。
- **服务对象**：C 端为普通/企业用户，B 端为单运营人员。
- **核心边界**：爬虫落库 → SQLite → FastAPI → 小程序 + admin-frontend；不包含多租户、复杂 RBAC、BI、客服等。

## 2. 快速开始

### 环境要求

- Python 3.9+
- Node.js（miniapp、admin-frontend）
- 微信开发者工具（仅小程序开发）
- 可选：Docker + Docker Compose（生产-like 部署）

### 启动步骤

**后端 API**

```bash
# 项目根目录
PYTHONPATH=. uvicorn server.main:app --reload --host 0.0.0.0 --port 8000
```

访问 [http://localhost:8000/docs](http://localhost:8000/docs) 查看 Swagger。

**小程序**

1. 微信开发者工具中打开 `miniapp/`，勾选「不校验合法域名…」以便本地联调。
2. 确认 `miniapp/src/config.ts` 中 `baseUrl: 'http://localhost:8000'`。
3. 开发时用 `npm run dev:weapp`（watch 模式，改代码自动重编译）；发布前用 `npm run build:weapp`。
4. 微信开发者工具中开启「修改时自动保存」或「编译时自动保存」，可减少手动刷新。

**运营后台**

```bash
cd admin-frontend && npm install && npm run dev
```

### 配置说明

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `NOTICES_DB` | `data/notices.db` | SQLite 路径 |
| `JWT_SECRET` | （见 server） | JWT 签名密钥，生产必须替换 |
| `ADMIN_TOKEN` | （见 server） | 管理端鉴权，生产必须替换 |

### 常见启动问题

- 数据库为空：可运行 site1/site2 的 backfill 或 incremental 拉取数据，或插入测试数据（见原《开发联调指南》）。
- 小程序请求失败：检查 baseUrl、关闭域名校验、确认后端已启动。

## 4. 上下游与外部依赖

- **数据源**：网站一（四川省公共资源交易平台）、网站二（四川政府采购网）等，见《原始数据接口文档》。
- **公网入口**：可选 Traefik（如 100.64.0.7），转发至 API（8000）与运营后台前端（8091）。
- **微信**：小程序发布需配置 request 合法域名指向后端 API 域名。

# 变更提案：小程序/前端接口文档

## Why

在实现小程序（及现有 Web 前端）前，需要一份**面向前端与小程序调用的接口规范**：明确「页面需要哪些接口、每个接口的路径/方法/参数/请求体/响应结构与字段含义」。当前仓库仅有《原始数据接口文档》——描述的是爬虫与数据源（网站一、网站二）之间的契约，而非前端与**自有后端**之间的契约。若缺少这份文档，小程序与 Web 开发将无法对齐数据契约、字段命名与筛选能力，联调与维护成本高。因此需要在实现 miniapp 之前，基于「原始数据能提供的字段」与「页面元素与交互需求」整理出一份**前端/小程序接口文档**。

## What Changes

- **新增一份接口文档**（建议命名如《接口文档-前端与小程序.md》或置于 `docs/` 下）：面向小程序与 Web 前端，描述其调用的**后端 API**（非原始数据源直连）。
- **文档内容**：接口清单（列表、详情、登录、注册、审核状态、收藏等）；每个接口的 Method、URL 或 path、请求参数/请求体、响应结构、字段定义与说明；与页面模块的对应关系（如首页列表、详情页、登录页所需字段）。
- **数据依据**：接口与字段设计需兼顾（1）《原始数据接口文档》中已落库或可落库的字段与数据源能力；（2）《PAGES_OVERVIEW》等页面功能对展示与筛选的需求。
- **不修改**：《原始数据接口文档》保持不变；本变更只新增「前端/小程序用」的接口规范，不实现后端代码。

## Capabilities

### New Capabilities

- **api-spec-doc**: 定义《接口文档-前端与小程序》的文档目标、适用范围、接口清单（列表/详情/登录/注册/审核/收藏等）及与页面的对应关系；规定文档必须包含的章节与每类接口必须描述的要素（Method、路径、参数、请求体、响应结构、字段表）。
- **api-spec-content**: 定义各接口的请求与响应契约：列表与详情接口的入参（分页、筛选、业务类型）、出参字段与类型；登录/注册/审核状态/收藏等接口的入参与出参；字段命名与含义与原始数据落库字段、页面展示需求的映射关系。

### Modified Capabilities

- 无。

## Impact

- **新增产物**：一份 Markdown 格式的接口文档（仓库内新文件），供小程序与 Web 前端实现与联调使用；miniapp-from-ui 变更中的 miniapp-data 规格将引用该文档。
- **依赖关系**：文档内容依赖《原始数据接口文档》与《PAGES_OVERVIEW》等现有文档，不改变爬虫或原始数据接口本身。
- **后续**：后端实现或 Mock 时可依据本接口文档实现；小程序与 ui 的请求封装以本文档为契约。

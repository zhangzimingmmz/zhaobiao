# 招投标数据源 — 接口规范文档

本文档基于《原始数据接口》中的**抓包请求与响应**整理而成，作为**爬虫系统设计**的正式依据：**哪些接口可用、请求/响应契约、每条响应中有哪些字段、各字段是否落库/仅展示/不采集**，均以本文档为准。抓包未覆盖的行为（如签名生成、详情 URL 生成规则）在文档中单独标明。

---

## 文档约定

- **接口可用性**：下表「爬虫可用接口清单」中的接口为当前设计允许使用的数据源；未列入的接口不视为契约的一部分。
- **字段采集建议**：
  - **落库**：爬虫应持久化，供列表/详情 API 与前端使用。
  - **仅展示**：可随响应带给前端展示，是否落库由存储设计决定。
  - **不采集**：爬虫不依赖、不落库；若响应中带出可丢弃或仅写 raw_json。
- **数据类型**：除注明「见说明」外，均为抓包中观测到的类型；若实际返回 null/缺失，爬虫应按可空处理。

# 一、网站一（四川省公共资源交易网）

- **Base URL**：`https://ggzyjy.sc.gov.cn`
- **列表**：三类业务（招标计划 002001009、招标公告 002001001、政府采购采购公告 002002001）共用同一列表接口，仅请求体中的 `condition` 不同。

---

## 1.1 网站一 - 列表接口

### 1.1.1 契约摘要

| 项 | 值 |
|----|-----|
| 接口标识 | site1-list |
| Method | POST |
| URL | `https://ggzyjy.sc.gov.cn/inteligentsearch/rest/esinteligentsearch/getFullTextDataNew` |
| Request Content-Type | `application/x-www-form-urlencoded; charset=UTF-8` |
| Request Body | 原始 JSON 字符串（非 form 键值对） |
| Response | JSON，顶层键 `result`；`result.totalcount` 为总条数，`result.records` 为当前页列表。 |

### 1.1.2 请求头（爬虫必须/建议）

| 请求头 | 必选 | 取值说明 |
|--------|------|----------|
| Content-Type | 是 | `application/x-www-form-urlencoded; charset=UTF-8` |
| User-Agent | 建议 | 浏览器或固定 UA，抓包中为 Chrome 145 |
| Origin | 建议 | `https://ggzyjy.sc.gov.cn` |
| Referer | 建议 | `https://ggzyjy.sc.gov.cn/jyxx/transactionInfo.html` |

### 1.1.3 请求体（JSON）

爬虫必须按下列结构组包，整体作为 **Body 原始字符串** 发送（不拆成 form 字段）。

| 字段 | 类型 | 必填 | 取值说明 |
|------|------|------|----------|
| token | string | 否 | 空字符串 `""` |
| pn | number | 是 | 分页偏移，从 0 开始 |
| rn | number | 是 | 每页条数，抓包为 12；探测时可用 1 |
| sdt | string | 否 | 空字符串 |
| edt | string | 否 | 空字符串 |
| wd | string | 否 | 空字符串 |
| inc_wd | string | 否 | 空字符串 |
| exc_wd | string | 否 | 空字符串 |
| fields | string | 否 | 空字符串 |
| cnum | string | 否 | 空字符串 |
| sort | string | 是 | JSON 字符串，如 `"{\"ordernum\":\"0\",\"webdate\":\"0\"}"` |
| ssort | string | 否 | 空字符串 |
| cl | number | 否 | 抓包为 10000 |
| terminal | string | 否 | 空字符串 |
| condition | array | 是 | 见下表「按业务类型的 condition」 |
| time | array | 是 | 时间条件，见下表 |
| highlights | string | 否 | 空字符串 |
| statistics | null | 否 | 字面 null |
| unionCondition | null | 否 | 字面 null |
| accuracy | string | 否 | 空字符串 |
| noParticiple | string | 否 | 抓包为 `"1"` |
| searchRange | null | 否 | 字面 null |

**time 数组格式（必填）**：

```json
[
  {
    "fieldName": "webdate",
    "startTime": "2026-03-14 00:00:00",
    "endTime": "2026-03-14 23:59:59"
  }
]
```

**按业务类型的 condition（必填，三选一）**：

| 业务 | categorynum | condition 数组（JSON） |
|------|-------------|-------------------------|
| 工程建设-招标计划 | 002001009 | `[{"fieldName":"categorynum","equal":"002001009","notEqual":null,"equalList":null,"notEqualList":null,"isLike":true,"likeType":2}]` |
| 工程建设-招标公告 | 002001001 | 同上，仅 `equal` 改为 `"002001001"` |
| 政府采购-采购公告 | 002002001 | `[{"fieldName":"ZHUANZAI","equal":"四川省政府采购网","notEqual":null,"equalList":null,"notEqualList":null},{"fieldName":"categorynum","equal":"002002001","notEqual":null,"equalList":null,"notEqualList":null,"isLike":true,"likeType":2}]` |

### 1.1.4 响应结构

- 顶层键：`result`（对象）。
- `result` 下键：`categorys`（数组）、`totalcount`（number）、`records`（数组）、`analyze`（数组）、`maxScore`（number）、`scorllId`（string）、`executetime`（string）。
- 爬虫**必须**使用的键：`result.totalcount`、`result.records`。
- 分页：**偏移制**。第一页 `pn=0`，第二页 `pn=rn`，依此类推。停止条件：`pn >= result.totalcount` 或当前页 `result.records` 为空数组。

| linkurl | string | **落库** | 详情页相对路径；绝对 URL = Base + linkurl |
| content | string | **落库** | 正文或摘要，可能为空字符串 |
| titlenew | string | 仅展示 | 与 title 可能相同，抓包中均有 |

# 二、网站二（四川政府采购网）

- **Base URL**：`https://www.ccgp-sichuan.gov.cn`
- **依赖**：列表接口需**验证码**（query 参数 `verifyCode`）与**动态签名头**（nsssjss、sign、time、url）；详情接口需签名与会话，不需验证码。验证码图片由本节的**获取验证码接口**返回，识别后填入列表的 `verifyCode`；签名生成方式**未在抓包中提供**，由爬虫侧外部模块提供。

---

## 2.0 网站二 - 获取验证码

### 2.0.1 契约摘要

| 项 | 值 |
|----|-----|
| 接口标识 | site2-get-verify |
| Method | GET |
| URL | `https://www.ccgp-sichuan.gov.cn/gpcms/rest/web/v2/index/getVerify` |
| Response Content-Type | `image/jpeg` |
| Response Body | 验证码图片二进制（JPEG） |

### 2.0.2 请求

- **Query 参数**：抓包中无 query 参数，直接 GET 即可。
- **请求头（建议）**：

| 请求头 | 取值说明 |
|--------|----------|
| Accept | `image/avif,image/webp,image/apng,image/svg+xml,image/*,*/*;q=0.8` 或 `image/*` |
| Referer | 与后续请求列表的页面一致，如 `https://www.ccgp-sichuan.gov.cn/maincms-web/noticeInformation?typeId=ggxx` 或 `https://www.ccgp-sichuan.gov.cn/maincms-web/massageListPage?typeId=gsxx-noticeType` |
| Cookie | 与列表请求共用同一会话，如 regionCode=510001; regionFullName=四川省; regionRemark=1 |

### 2.0.3 响应

- **HTTP 200** 时：响应体为**验证码图片**（Content-Type: image/jpeg），无 JSON。
- 爬虫或外部模块需对该图片进行**识别**（如 OCR 或打码平台），得到 4 位数字字符串后，填入**采购意向公开列表**或**采购公告列表**的 Query 参数 `verifyCode` 中。
- 验证码与当前会话（Cookie）绑定，同一会话下先调本接口取图、识别，再带该 Cookie 与 `verifyCode` 调列表接口。

### 2.0.4 与列表接口的关系

调用顺序建议：1）GET getVerify（拿到图片 + 保持 Cookie）；2）识别图片得到 verifyCode；3）同一 Cookie 下调用 selectInfoForIndex，并传入 verifyCode 与签名头。

## 2.1 网站二 - 采购意向公开 - 列表

### 2.1.1 契约摘要

| 项 | 值 |
|----|-----|
| 接口标识 | site2-list-intention |
| Method | GET |
| URL | `https://www.ccgp-sichuan.gov.cn/gpcms/rest/web/v2/info/selectInfoForIndex` |

### 2.1.2 Query 参数

| 参数 | 必填 | 取值说明 |
|------|------|----------|
| currPage | 是 | 页码，从 1 开始 |
| pageSize | 是 | 每页条数，抓包为 10 |
| siteId | 是 | 固定 `94c965cc-c55d-4f92-8469-d5875c68bd04` |
| channel | 是 | 固定 `c5bff13f-21ca-4dac-b158-cb40accd3035` |
| noticeType | 是 | 采购意向公开为 `59` |
| title | 否 | 空或关键词 |
| purchaseManner, openTenderCode, purchaser, agency, purchaseNature | 否 | 空或筛选值 |
| operationStartTime | 否 | 时间范围起，如 "2026-03-14 00:00:00" |
| operationEndTime | 否 | 时间范围止 |
| verifyCode | 是 | 验证码，由外部模块提供 |
| _t | 建议 | 毫秒时间戳，与请求头 time 一致 |

### 2.1.3 请求头（爬虫必须/建议）

| 请求头 | 必选 | 取值说明 |
|--------|------|----------|
| Referer | 建议 | `https://www.ccgp-sichuan.gov.cn/maincms-web/massageListPage?typeId=gsxx-noticeType` |
| nsssjss | 是 | 动态签名，由外部模块提供 |
| sign | 是 | 动态签名 |
| time | 是 | 毫秒时间戳，与 _t 一致 |
| url | 是 | 请求 path：`/gpcms/rest/web/v2/info/selectInfoForIndex`（不含域名与 query） |
| Cookie | 建议 | 抓包中为 regionCode=510001; regionFullName=四川省; regionRemark=1 |

### 2.1.4 响应结构

- 顶层键：`code`（string）、`msg`（string）、`data`（object）。
- `data` 下键：`rows`（数组）、`total`（number）。爬虫**必须**使用 `data.rows`、`data.total`。
- 分页：**页码制**。总页数 = ceil(data.total / pageSize)。停止条件：currPage 超过总页数或当前页 `data.rows` 为空。

### 2.1.5 响应 data.rows 单条字段与采集建议

| 字段 | 类型 | 采集建议 | 说明 |
|------|------|----------|------|

# 三、依赖与未约定项（抓包未提供）

以下为**未在抓包中约定且会阻塞接入**的项，爬虫实现前必须先行解决或约定：

| 站点 | 阻塞项 | 说明 |
|------|--------|------|
| 网站二 | 签名头（nsssjss、sign、time、url） | 列表与详情接口均依赖上述请求头，生成规则未在抓包中暴露。**无有效签名则请求会被拒绝。** 实现时需确认：签名是否与请求 URL 的 query 或完整 URL 有关，并按实际规则生成。 |

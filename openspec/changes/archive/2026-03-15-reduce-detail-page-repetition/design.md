# Design: 减少招投标详情页重复内容

## Context

- **现状**：招投标详情页分为三块——信息卡片（分类、标题、发布时间、查看原文）、关键信息（发布时间、项目名称、预算等）、公告正文。信息卡片与关键信息在「发布时间」「项目名称」上完全重复（projectName 与 title 同源）。
- **数据**：后端 `projectName` 直接使用 `row["title"]`，二者等价。

## Goals / Non-Goals

**Goals:**
- 消除关键信息与信息卡片之间的重复展示
- 关键信息仅展示卡片未覆盖的字段

**Non-Goals:**
- 不处理公告正文（content）内部的重复——来自原始 HTML，解析裁剪风险大
- 不修改 API 或后端字段

## Decisions

### 1. 排除字段

**决策**：从 `structuredRows` 中排除「发布时间」「项目名称」。二者已在 `detail-card` 中通过 `detail.publishTime`、`detail.title` 展示。

**实现**：在构建 `structuredRows` 的数组中，不包含 `['发布时间', detail.publishTime]` 与 `['项目名称', detail.projectName]`。其余字段（预算金额、地点、招标人/采购人、代理机构、报名开始、报名截止、开标时间）保留。

### 2. 信息详情页

**决策**：信息详情页当前无「关键信息」区块，仅有 head（标题、发布时间）、摘要、正文，无重复。本变更仅修改招投标详情页。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 用户期望在关键信息中看到项目名称 | 项目名称已在卡片标题显著展示，移除后仍可读 |

# 后台管理接口文档

本文档是统一运营后台的正式接口入口，覆盖企业审核、企业目录、采集控制、运行记录以及当前已识别但尚未完成的后台接口缺口。当前仓库中的极简运营后台前端工程位于 `admin-frontend/`。

## 1. 认证方式

当前后台管理接口支持以下两种管理员认证方式：

```http
Authorization: Bearer <ADMIN_TOKEN>
```

说明：

- `ADMIN_TOKEN` 由服务端环境变量配置
- 也支持通过 `POST /api/admin/login` 获取管理员 JWT
- 当前实现是轻量后台账号方案：1 个环境变量 `super_admin` + 多个数据库 `reviewer`
- 与小程序用户的 `/api/auth/login` 不是同一套认证逻辑

## 2. 接口状态说明

本文档使用以下状态标记：

- `已实现`：后端已有真实接口和代码实现
- `部分实现`：可由现有接口拼接完成，但没有独立聚合接口
- `未完成`：需求已明确，但当前后端没有独立接口实现

## 3. 后台登录相关接口

### 3.1 管理员登录

- 接口：`POST /api/admin/login`
- 状态：`已实现`

说明：

- `super_admin` 使用环境变量账号密码登录
- `reviewer` 使用后台管理页面创建后的数据库账号登录
- 默认超管账号由 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 控制
- `ADMIN_REVIEWER1_*`、`ADMIN_REVIEWER2_*` 仅作为兼容导入入口，不再作为日常维护方式
- 登录成功后返回管理员 JWT，供前端本地长期持久化

请求体：

```json
{
  "username": "admin",
  "password": "your-password"
}
```

响应体：

```json
{
  "code": 200,
  "data": {
    "token": "admin-jwt-token",
    "tokenType": "Bearer",
    "username": "admin",
    "adminId": "super_admin",
    "role": "super_admin"
  }
}
```

补充说明：

- `reviewer` 登录成功时 `role=reviewer`
- 企业审核的 `audited_by` 会写入当前登录管理员的真实 `adminId`
- 企业档案编辑、企业数据删除、reviewer 账号管理仅允许 `super_admin`

### 3.2 审核员管理

- 接口：`GET /api/admin/reviewers`
- 接口：`POST /api/admin/reviewers`
- 接口：`POST /api/admin/reviewers/{admin_id}/reset-password`
- 接口：`POST /api/admin/reviewers/{admin_id}/status`
- 状态：`已实现`

说明：

- 仅 `super_admin` 可访问
- 支持 reviewer 列表查看、创建、密码重置、启用/禁用
- reviewer 禁用后不可继续登录，但历史审核记录仍保留原审核人展示

创建请求体：

```json
{
  "username": "reviewer3",
  "password": "reviewer-pass-123"
}
```

状态更新请求体：

```json
{
  "status": "disabled"
}
```

### 3.2 后台总览接口

- 接口：`GET /api/admin/dashboard`
- 状态：`部分实现`

说明：

- 当前阶段不新增独立 Dashboard 聚合接口
- 当前总览页采用前端聚合策略
- 前端通过以下接口组合得到总览数据：
  - `GET /api/admin/reviews?status=pending&page=1&pageSize=1`
  - `GET /api/admin/crawl/runs?limit=1`
- 如后续需要更复杂的总览，可再补 `GET /api/admin/dashboard`

## 5.5 文章管理接口

- 状态：`已实现`
- 说明：运营人员在后台创建、编辑、发布、下线、删除文章；支持公众号链接校验、自动提取标题/封面/摘要、重复检查；操作记录写入 article_operation_logs

主要接口：

| 接口 | Method | 说明 |
|------|--------|------|
| 文章列表 | GET /api/admin/articles | 分页、状态筛选(draft/published)、分类筛选、关键词搜索 |
| 文章详情 | GET /api/admin/articles/:id | 单篇文章详情 |
| 创建文章 | POST /api/admin/articles | 创建草稿 |
| 更新文章 | PUT /api/admin/articles/:id | 更新草稿 |
| 发布 | POST /api/admin/articles/:id/publish | 发布文章 |
| 下线 | POST /api/admin/articles/:id/unpublish | 下线文章 |
| 删除 | DELETE /api/admin/articles/:id | 删除文章 |
| 校验链接 | POST /api/admin/articles/validate-url | 校验公众号链接并提取信息 |
| 检查重复 | POST /api/admin/articles/check-duplicate | 检查链接是否已存在 |
| 操作日志 | GET /api/admin/articles/:id/logs | 查询文章操作历史 |

## 6. 采集控制接口

### 6.1 获取可执行动作列表

- 接口：`GET /api/admin/crawl/actions`
- 状态：`已实现`

响应说明：

- 返回允许在后台触发的受控动作白名单
- 当前不允许任意脚本和任意 CLI 参数透传

支持动作：

| Action Key | Site | Kind | Notes |
|---|---|---|---|
| `site1.incremental` | site1 | routine | 无额外参数 |
| `site1.recovery` | site1 | routine | 无额外参数 |
| `site1.backfill` | site1 | maintenance | 需要 `start` / `end`，可选 `category` |
| `site2.incremental` | site2 | routine | 无额外参数 |
| `site2.recovery` | site2 | routine | 无额外参数 |
| `site2.backfill` | site2 | maintenance | 需要 `start` / `end`，可选 `noticeType` |
| `site2.precheck` | site2 | diagnostic | 无额外参数 |
| `site2.reconcile` | site2 | diagnostic | 需要 `start` / `end` |

明确拒绝的动作：

- `site2.cleanup`
- `site2.backfill-formal`
- `site2.reconcile.verify-recovery`
- `site2.reconcile.verify-boundary`

### 6.2 提交采集运行请求

- 接口：`POST /api/admin/crawl/runs`
- 状态：`已实现`

请求体：

```json
{
  "actionKey": "site2.backfill",
  "params": {
    "start": "2026-03-01",
    "end": "2026-03-07"
  }
}
```

成功响应示例：

```json
{
  "code": 200,
  "message": "请求已受理",
  "data": {
    "id": "run-id",
    "site": "site2",
    "actionKey": "site2.backfill",
    "status": "queued"
  }
}
```

说明：

- 请求是异步的
- 若站点已有 `queued` 或 `running` 的后台任务，请求会被拒绝并记录为 `rejected`

## 7. 运行记录接口

### 7.1 获取运行记录列表

- 接口：`GET /api/admin/crawl/runs`
- 状态：`已实现`

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| status | string | 否 | 运行状态 |
| site | string | 否 | 站点标识 |
| actionKey | string | 否 | 动作键 |
| limit | number | 否 | 默认 50，最大 200 |

说明：

- 当前列表接口返回 `list`
- 主要字段包括 `id`、`site`、`actionKey`、`status`、`statusReason`、`summary`、`requestedAt`

### 7.2 获取运行详情

- 接口：`GET /api/admin/crawl/runs/{run_id}`
- 状态：`已实现`

当前详情可返回的主要字段：

- `id`
- `site`
- `taskName`
- `actionKey`
- `runKind`
- `requestedBy`
- `status`
- `statusReason`
- `requestPayload`
- `resultPayload`
- `summary`
- `requestedAt`
- `startedAt`
- `finishedAt`
- `logPath`
- `command`
- `exitCode`
- `fetchedCount`
- `upsertedCount`
- `errorCount`

说明：

- 详情页已经能支撑当前极简后台查看动作、参数、执行摘要和失败原因
- 更完整的日志展示能力仍可后续增强

## 8. 权限与约束说明

- 所有 `/api/admin/*` 接口都要求有效管理员 token
- 普通用户 token 不能访问管理员接口
- 当前后台仍然是最小内部使用模式，不是正式多角色系统
- 采集控制只允许受控白名单动作
- 后台请求不会直接执行任意 Python 模块或任意 CLI 参数

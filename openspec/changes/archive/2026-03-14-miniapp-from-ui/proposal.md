# 变更提案：基于现有 UI 实现微信小程序

## Why

现有招投标信息平台前端为 Web 版（`ui/`，React + Vite），仅能在浏览器使用。需要提供微信小程序端，使用户可在微信内直接使用同一套业务（招投标/政府采购/信息展示、登录注册、收藏与详情），并复用现有接口与设计规范。将小程序实现放在仓库内 `miniapp/` 目录，便于与 Web、爬虫、接口文档统一维护。

## What Changes

- 在 **`miniapp/`** 下新建 Taro（React）微信小程序项目，产出可被微信开发者工具打开的编译目录（如 `miniapp/dist`）。
- **7 个页面**与 `ui/` 对应：首页（含一级/二级菜单与筛选）、招投标详情、信息展示详情、登录、注册/企业认证、审核状态、我的。
- **组件与视觉**：复用 `ui/` 的设计系统（政务蓝主色、字号与圆角等），用 Taro 组件与 WXSS 实现等价交互与布局。
- **数据与接口**：对接现有后端接口（与 `接口文档.md` 一致），实现列表、详情、登录/注册、收藏等能力；不新增后端接口。
- **工程**：支持 `npm run dev:weapp` / `build:weapp`，产出目录可用微信开发者工具「导入项目」打开。

## Capabilities

### New Capabilities

- **miniapp-project**: 在 `miniapp/` 下初始化 Taro（React）项目、配置编译为微信小程序、构建脚本与可在微信开发者工具中打开的产出目录。
- **miniapp-pages**: 7 个页面与路由（首页、招投标详情、信息展示详情、登录、注册、审核状态、我的），页面层级与 `ui/PAGES_OVERVIEW.md` 一致。
- **miniapp-ui**: 小程序端组件与设计规范（顶部栏、一级/二级 Tab、筛选栏与弹层、列表卡片、底部导航等），视觉与交互与 `ui` 设计系统一致。
- **miniapp-data**: 小程序端与现有后端接口对接（列表、详情、登录/注册、收藏等），数据契约遵循 `接口文档.md`，不修改后端。

### Modified Capabilities

- 无（不修改现有 spec 行为）。

## Impact

- **新增目录**：`miniapp/`（源码 + 依赖），以及构建产出目录（如 `miniapp/dist`）。
- **依赖**：`miniapp/` 内新增 Taro、React 及小程序相关依赖；仓库根可选用 pnpm workspace 统一管理。
- **接口与后端**：不新增或修改后端接口；小程序与 Web 共用同一套 API 与数据契约。
- **文档**：可在 `miniapp/README.md` 中说明本地构建与用微信开发者工具打开的步骤。小程序与前端请求契约以 **`docs/接口文档-前端与小程序.md`** 为准。

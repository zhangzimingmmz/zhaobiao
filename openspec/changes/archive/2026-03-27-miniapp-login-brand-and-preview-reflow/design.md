## Context

当前登录页已经具备正确的门禁、审核状态判断和游客预览能力，但页面表达仍停留在“说明页 + 表单页 + 预览页”的堆叠模式。具体问题包括：

- `TopBar` 标题 `登录`、eyebrow `登录`、主标题 `使用已审核账号登录` 同时出现，造成重复命名。
- 说明文案过长，导致首屏文字密度过高，表单没有成为绝对主角。
- 登录页缺少品牌承接区，和参考稿里的“品牌名 / 轻视觉背景 / 表单”关系不一致。
- “最新招采预览”距离表单过近，被误读为首屏主内容而不是辅助内容。
- 现有长期 spec 中仍包含早期的“移除去注册按钮”“手机号验证码登录”等旧约束，需要与当前账号密码登录流和注册入口现实重新对齐。

这次 change 只处理登录页的视觉骨架与文案层级，不改登录逻辑、审核判断、预览数据来源或点击拦截行为。

## Goals / Non-Goals

**Goals:**

- 让登录页只保留一个清晰的页面标题来源，消除重复 `登录` 表达。
- 将说明区改造成轻品牌区，让页面更接近参考稿的首屏关系。
- 显著减少首屏文字量，使表单成为最重要的视觉块。
- 将“最新招采预览”下移为次级区域，降低其对首屏的干扰。
- 校正长期 spec 中与当前登录形态不一致的旧要求。

**Non-Goals:**

- 不更换登录方式，不改成手机号验证码，也不改动任何后端接口。
- 不移除当前注册入口、协议文案、预览数据和点击拦截逻辑。
- 不引入远程品牌素材或新的网络依赖。
- 不同时重做注册页、审核页或其他页面。

## Decisions

### 1. 登录页首屏改为“单标题 + 品牌区 + 表单卡”

**选择**：保留 `TopBar` 中的 `登录` 作为唯一页面标题，在正文中删除 eyebrow `登录`，并用品牌区取代现在的说明型 intro。

**结构：**

```text
TopBar(登录)
↓
品牌区(logo/字标 + 一句短提示)
↓
表单卡
↓
注册入口 + 协议
↓
预览区
```

**理由**：

- `TopBar` 已经承担页面命名，不需要正文再重复一层 `登录`。
- 参考稿更强调品牌识别和表单主导，而不是大段说明文字。

**替代方案**：保留主标题 `使用已审核账号登录`，只删除 eyebrow。

- 被否决原因：仍会让页面上半部偏说明化，不够接近参考稿。

### 2. 品牌区采用本地轻素材或纯样式承接，不依赖远程资源

**选择**：品牌区允许使用本地静态 logo 素材，也允许先用纯色块/渐变 + 品牌文字实现；不得引入依赖网络的远程图。

**理由**：

- 登录页是首次进入的关键页，不能依赖网络拉品牌图。
- 当前小程序已有本地资源体系，新增一个轻品牌素材或纯 CSS 容器即可满足目标。

**替代方案**：直接复用 launch 页夜景图或远程背景图。

- 被否决原因：夜景图过重，会把登录页重新推回海报页；远程图会增加首屏不稳定性。

### 3. 预览区保留，但降级为第二层内容并减少文字

**选择**：继续保留“最新招采预览”与真实数据轮播，但通过更大的垂直间距把它下移，并删除额外的解释文字。

**要求**：

- 预览区标题保留。
- `登录后可查看完整详情与全部数据页面` 这类重复说明应删除或显著弱化。
- 卡片内 `点击后需登录查看详情` 不再占一整行静态文案，点击提示由现有 toast 承担。

**理由**：

- 审核和游客预览策略仍然有效，不需要删除预览。
- 真正的问题是预览离表单太近、文案太多，而不是预览本身存在。

**替代方案**：完全删除预览区。

- 被否决原因：不符合当前审核策略，也会让游客态内容过空。

### 4. 登录页长期 spec 以当前账号密码登录流为准

**选择**：在 `miniapp-login-ui` delta 中明确：

- 注册入口可以保留，但必须是低权重文本入口，不再是误导性的并列主 CTA。
- 登录页围绕账号名 + 密码表单组织，而不是历史上的手机号 + 验证码描述。

**理由**：

- 长期 spec 当前已经落后于产品现实，如果不纠正，后续 archive 会持续把错误约束写回主线。

**替代方案**：只改视觉，不动 spec。

- 被否决原因：会让未来变更继续建立在错误的登录约束上。

## Risks / Trade-offs

- [Risk] 品牌区做得过重会再次把登录页推向宣传页 → Mitigation: 品牌区只做轻承接，不使用大图海报或高饱和渐变。
- [Risk] 预览区下移后，长屏设备上底部可能更空 → Mitigation: 通过品牌区和表单卡的更稳定关系来撑住首屏，预览区作为第二层补充即可。
- [Risk] 修改 `miniapp-login-ui` 会与早期 archive 的历史语义不同 → Mitigation: 在 delta spec 中明确这是对旧登录形态的纠偏，保持当前真实产品为准。

## Migration Plan

1. 更新 `miniapp-auth-page-family` 和 `miniapp-login-ui` 的 delta spec，统一新的登录页骨架和交互边界。
2. 调整 `miniapp/src/pages/login/index.tsx`，删除重复标题层，增加品牌区结构，减少预览区文案。
3. 调整 `miniapp/src/pages/login/index.scss` 与必要的共享样式，完成品牌区、表单卡、预览区的重新排布。
4. 重新编译小程序并在微信开发者工具中截图，对照参考稿确认：
   - 不再出现两个“登录”
   - 首屏文字显著减少
   - 预览区明显下移
   - 表单成为首屏主角

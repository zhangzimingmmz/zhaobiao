## Context

当前小程序首页承担了过多职责：品牌说明、搜索引导、频道入口、平台说明都堆在一个页面里，因此视觉层级分散，也和目标稿的“深蓝品牌横幅 + 三个大入口”结构不一致。现有首页代码集中在 `miniapp/src/pages/index/index.tsx` 和 `miniapp/src/pages/index/index.scss`，频道跳转本身已经独立，不需要再让首页承载搜索与说明逻辑。

这次变更只重构首页入口层，不调整登录态、频道页内部结构和后端接口。约束是：仍使用当前小程序的白蓝轻量 design tokens 与 `TopBar` 体系，同时让首页在结构上更接近目标稿，而不是继续叠加平台首页能力。

## Goals / Non-Goals

**Goals:**
- 将首页重构为纯频道入口页，建立“品牌横幅 + 三入口”的首屏结构。
- 让首页顶部品牌区成为唯一主视觉，三张入口卡成为唯一主要交互区。
- 删除当前首页上的搜索入口、频道副标题说明与平台说明区，避免页面继续承担“信息平台首页”的混合职责。
- 保留现有频道跳转关系，不改变 `工程建设 / 政府采购 / 信息公开` 的页面归属。

**Non-Goals:**
- 不新增搜索功能或新的信息流模块。
- 不修改频道页、详情页、收藏页和认证页结构。
- 不引入新的 API、数据源或导航目的地。

## Decisions

### 决策 1：首页改为纯入口页，而不是继续保留平台首页结构
选择删除搜索入口条和平台说明带，只保留品牌横幅与三个频道入口卡。

原因：
- 目标稿是明确的频道入口页，不是搜索首页。
- 当前首页之所以显得“像静态菜单页又像平台页”，就是因为混合了承担不同任务的模块。
- 搜索和平台说明保留在首页会持续分散视觉焦点，无法收敛成目标结构。

备选方案：
- 保留搜索条，只做视觉重构。
  - 否决原因：这会继续让页面承担平台首页角色，结构上仍然不对齐目标稿。

### 决策 2：顶部采用重品牌横幅，而不是弱 hero 文案
选择把当前 `TopBar + hero copy` 重构为深蓝品牌横幅，包含品牌标题、副标题和轻装饰图形。

原因：
- 目标稿的顶部不是“白底标题 + 小文案”，而是一块强品牌头图。
- 现有首页主焦点不足，必须通过头图建立页面唯一主视觉。
- 头图可以继续复用全站蓝色 token，不需要引入图片资源。

备选方案：
- 保留现有白底 hero，只微调字体和间距。
  - 否决原因：无法从结构上形成目标稿那种“上重下轻”的入口页形态。

### 决策 3：入口卡改为大入口按钮卡，不再携带说明性副文案
选择三张卡仅保留彩色图标块、频道标题和右箭头，不再展示频道副标题和推荐 badge。

原因：
- 目标稿的三张卡本质上是主导航按钮，不是说明卡。
- 现有副标题和 badge 会把卡片拉回“信息卡 / 说明卡”风格。
- 首页当前没有其他重内容模块，入口卡必须更厚重、更直接。

备选方案：
- 保留副标题，删掉 badge。
  - 否决原因：副标题仍会让卡片重心下沉，无法达到目标稿那种大入口按钮感。

### 决策 4：长屏留白通过入口页比例解决，而不是再补说明带
选择通过放大头图和入口卡、压缩页面结构层级来解决长屏空白，而不是继续添加底部说明内容。

原因：
- 平台说明带只是临时承接空白，不是目标结构的一部分。
- 首页改成纯入口页后，长屏节奏应该由“头图 + 三卡”的比例来解决。
- 这也避免未来首页继续滑回“菜单页 + 说明页”的混合状态。

## Risks / Trade-offs

- [首页删除搜索入口后，平台感会弱于信息平台首页] → 这是有意取舍；如果后续仍需要搜索，应在频道页或专门的搜索页承载。
- [去掉副标题后，三个入口的业务边界不再通过首页解释] → 依赖更清晰的频道标题、图标与频道页内部结构承接。
- [重品牌横幅若做得过重，可能与全站白蓝轻量体系脱节] → 仅在首页顶部使用深蓝横幅，其他区域继续使用全站 white-blue tokens 和轻卡片体系。

## Context

当前首页组件结构如下:

```text
TopBar
PrimaryTabs
SecondaryTabs
FilterBar
List
```

代码层面 `PrimaryTabs`、`SecondaryTabs`、`FilterBar` 都各自带背景:

- `PrimaryTabs` 自带频道轨道背景
- `SecondaryTabs` 自带二级胶囊区域背景
- `FilterBar` 还是一张完整白底卡片

这会把本来连续的控制区切成多段。用户已经多次反馈筛选区“宽”“高”“像另一套组件”,而这些现象的共同上游都是容器关系碎片化。方向 2 的核心判断是:

- 一级频道是页面级导航,应保持独立。
- 一级频道下面的当前频道控制区,才应该是一个统一的操作面板。

因此这个 change 不追求把一级频道也塞进大面板,而是把“二级切换 + 公告类型切换 + 搜索 + 筛选按钮”收敛成同一个背景容器。

## Goals / Non-Goals

**Goals:**

- 为首页控制区建立清晰的两层结构: 页面级频道导航层 + 当前频道控制面板层。
- 让 `SecondaryTabs`、`FilterBar` 以及工程建设状态下的公告类型切换在视觉上归属于同一个面板。
- 让不同业务态继续保留各自支持的控件集,但都共享一致的面板容器语言。
- 降低后续通过单独调按钮高宽来修复首页协调性的需求。

**Non-Goals:**

- 本 change 不重做一级频道 `PrimaryTabs` 的交互模型。
- 本 change 不改变筛选条件本身的业务含义、接口参数或弹层内容。
- 本 change 不要求所有业务态拥有完全相同的控件数量或完全相同的内部排布。
- 本 change 不在本阶段重新设计列表卡片、详情页或底部导航。

## Decisions

### 1. 首页顶部控制区采用“两层归属”而不是“整段大卡片”

首页顶部 SHALL 被建模为两层:

```text
TopBar
PrimaryTabs
HomeControlPanel
  ├─ SecondaryTabs
  ├─ announcement-type segment (when supported)
  ├─ Search
  └─ Filter buttons
List
```

也就是说,一级频道导航保持独立,其下的当前频道控制区统一进入 `HomeControlPanel`。

备选方案:

- 把 `PrimaryTabs` 也并入同一面板: 被否决,因为一级频道更接近页面级导航,独立存在时语义更清楚。
- 保持现状,继续只调子组件样式: 被否决,因为已被多轮反馈证明无法根治顶部控制区的割裂感。

### 2. 统一背景由页级面板容器提供,子组件去卡片化

`HomeControlPanel` 负责:

- 背景色或渐变底
- 外层圆角
- 外层边框
- 外层阴影
- 面板内统一留白

子组件 SHALL 去掉“再起一张完整白底卡片”的做法:

- `SecondaryTabs` 只保留标签/胶囊本身的态样式,不再承担一整块背景。
- `FilterBar` 外层不再表现为独立白卡,只保留搜索框、按钮组和必要内部轻底色。
- 工程建设态的公告类型切换作为面板内部一个 section,而不是独立于面板的另一张卡。

备选方案:

- 保留子组件各自背景,再外面套一层背景: 被否决,因为会形成“卡中卡”的重复层级。

### 3. 面板内部保留层次,但靠 section 节奏而不是多层卡片

统一背景不意味着内部失去层次。面板内部仍应通过 section 间距、轻底、描边、按钮态来区分:

- 二级 tab 区
- 公告类型切换区
- 搜索区
- 筛选按钮区

信息展示态因为控件更少,可以直接显示为更轻的面板版本,但依然属于同一控制面板家族。

备选方案:

- 所有 section 都做完全同样的白底小卡: 被否决,因为会重新引入碎片化。

### 4. 控件顺序保持现有业务结构,不因面板合并而改业务层级

虽然视觉上并入同一面板,但业务层级仍保持:

- 一级频道切换决定当前业务域
- 二级切换决定当前列表态
- 公告类型切换仅在支持的工程建设态出现
- 搜索与筛选只作用于当前列表态

这保证本 change 是视觉与容器结构重构,而不是业务逻辑改造。

## Risks / Trade-offs

- [Risk] 面板统一后首屏可能显得更“成块” → Mitigation: 保持一级频道独立,避免把全部控件压成一整块重卡片。
- [Risk] 信息展示态控件较少,若套用工程/采购同等密度会显得空 → Mitigation: 允许信息展示态使用同一家族但更轻的 section 间距和内部留白。
- [Risk] 若只改页级容器、不清理子组件背景,会出现卡中卡 → Mitigation: 任务中要求同步梳理 `PrimaryTabs`、`SecondaryTabs`、`FilterBar` 的背景职责。
- [Risk] 后续又回到“继续调单个按钮尺寸” → Mitigation: 在任务里把容器职责和视觉验收放在按钮微调之前。

## Verification Approach

1. 对工程建设、政府采购、信息展示三个一级频道分别验证:
   - 一级频道是否仍独立于控制面板
   - 二级切换、搜索、筛选是否归入同一背景区域
2. 对工程建设态验证公告类型切换是否也属于同一面板。
3. 对采购公告态验证两行筛选按钮虽然可换行,但仍处于同一控制面板中。
4. 对信息展示态验证面板密度更轻,但不再显示割裂的独立白卡。
5. 以真机或开发者工具截图检查首页顶部是否从“多段散装控件”收敛为“一级导航 + 统一控制面板”的两层结构。

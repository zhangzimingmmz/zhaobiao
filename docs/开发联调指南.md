# 开发联调指南

本文档说明如何在本地开发环境中让小程序与后端 API 联调。

## 前置条件

- Python 3.9+（已安装）
- 微信开发者工具（已安装并登录）
- 已安装项目依赖：`pip3 install fastapi uvicorn[standard] python-jose[cryptography] passlib requests`

---

## 第一步：关闭微信开发者工具的域名校验

微信小程序默认会校验请求域名是否在「合法域名」列表中。本地开发时 `http://localhost:8000` 不在列表里，需关闭校验：

1. 打开微信开发者工具
2. 右上角点击「详情」
3. 切换到「本地设置」标签
4. 勾选 ✅ **「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**
5. 重新编译项目（`Ctrl+B` 或顶部菜单「编译」）

> ⚠️ 此设置仅在开发模式生效，不影响生产包。上线前需在微信公众平台后台配置合法域名（HTTPS）。

---

## 第二步：启动本地后端服务

在项目根目录执行：

```bash
PYTHONPATH=. python3 -m uvicorn server.main:app --reload --port 8000
```

服务启动后，访问 [http://localhost:8000/docs](http://localhost:8000/docs) 可查看所有接口的 Swagger 文档。

### 环境变量（可选）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `NOTICES_DB` | `data/notices.db` | SQLite 数据库路径 |
| `JWT_SECRET` | `zhaobiao-dev-secret-2026` | JWT 签名密钥（生产环境务必替换） |

---

## 第三步：确认小程序 baseUrl 配置

打开 `miniapp/src/config.ts`，确认：

```ts
export const config = {
  baseUrl: 'http://localhost:8000',
}
```

---

## 第四步：准备测试数据（如数据库为空）

如果 `data/notices.db` 中没有数据，首页列表会显示「暂无数据」。可通过以下方式插入测试数据：

**方式一：运行爬虫（需网络访问数据源）**
```bash
cd crawler
python3 -m site1.tasks.backfill --start 2026-03-01 --end 2026-03-14
```

**方式二：手动插入 SQLite 测试记录**
```bash
sqlite3 data/notices.db "INSERT INTO notices (site, id, title, publish_time, category_num) VALUES ('site1', 'test-001', '测试招标公告', '2026-03-14 10:00:00', '002001001');"
```

## 联调验证清单

完成上述步骤后，在微信开发者工具的「调试器」→「Network」面板确认：

- [ ] 首页加载时有 `GET /api/list` 请求，返回 HTTP 200
- [ ] 选择地区筛选后，请求参数中包含 `regionCode`
- [ ] 选择采购方式筛选后，请求参数中包含 `purchaseManner`
- [ ] 点击列表项跳转详情页，有 `GET /api/detail/bid/:id` 请求
- [ ] 登录成功后 token 存入 storage
- [ ] profile 页显示真实企业信息

## 常见问题

### Q: 仍然报「request 合法域名校验出错」

检查是否已重新编译。有时需要在开发者工具中点击「清缓存」→「清除编译缓存」后再编译。

### Q: 接口返回 401

token 已过期（默认 7 天）。重新获取验证码并登录，token 会自动存入 storage。

### Q: 首页显示「加载失败」而非数据

后端服务可能未启动，或端口不是 8000。检查终端中 uvicorn 是否在运行。

# miniapp-ui 规格

## ADDED Requirements

### Requirement: 视觉与设计系统与 ui 一致

小程序端主色、中性色、功能色、字号与圆角等 MUST 与 `ui/DESIGN_SYSTEM.md`（或同等设计规范）一致。主色为政务蓝 #1677FF；成功/警告/错误色及微信绿（#07C160）用于对应场景；信息层级与密度与 Web 端设计保持一致。

#### Scenario: 主色与关键组件使用正确色值

- **WHEN** 用户查看任意页面的主要按钮、选中态 Tab、链接等
- **THEN** 使用的颜色与设计系统中的主色、主色深（hover/按下）等一致

#### Scenario: 信息展示详情页公众号按钮为微信绿

- **WHEN** 用户进入信息展示详情页并查看「公众号原文」按钮
- **THEN** 按钮使用设计系统中的微信绿（#07C160）

### Requirement: 核心组件与布局等价实现

小程序 MUST 提供与 ui 对应的核心组件与布局：顶部栏（TopBar）、一级 Tab（PrimaryTabs）、二级 Tab（SecondaryTabs）、筛选栏（FilterBar）、筛选弹层（FilterSheet）、招投标卡片（BidCard）、信息展示卡片（InfoCard）、底部导航（BottomNav）、空状态（EmptyState）等。交互（点击、切换、弹层开闭）MUST 与 PAGES_OVERVIEW 及设计系统描述一致。

#### Scenario: 顶部栏与底部导航表现一致

- **WHEN** 用户在任意具备顶部栏或底部导航的页面查看
- **THEN** 标题、返回/个人中心/收藏等入口位置与行为与 ui 描述一致；底部导航在首页/我的间切换

#### Scenario: 筛选弹层可打开并选择条件

- **WHEN** 用户在首页点击筛选区域触发筛选面板
- **THEN** 以弹层形式展示当前菜单对应的筛选项（如发布时间、地区、采购方式等），用户可选择并关闭，筛选结果反映到列表

### Requirement: 列表与卡片信息层级清晰

列表项（招投标卡片、信息展示卡片）MUST 展示与 ui 一致的关键字段（如标题、类型标签、时间、摘要等），信息层级与密度符合设计系统，触摸区域与可点击范围适合小程序环境。

#### Scenario: 招投标列表卡片展示关键信息

- **WHEN** 用户在首页浏览招投标列表
- **THEN** 每条卡片展示公告类型、标题、发布时间等关键信息，布局与设计规范一致

#### Scenario: 无数据时显示空状态

- **WHEN** 列表无数据或加载失败
- **THEN** 展示空状态组件（或等价提示），与 ui 的 EmptyState 意图一致

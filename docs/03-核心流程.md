# 03-核心流程

> **作用**：说明业务如何流动、状态如何流转、异常如何补偿。  
> **给谁看**：开发、测试、交接人、业务熟悉者、reviewer。  
> **什么时候改**：流程步骤、状态流转、接口顺序、审核规则或补偿逻辑变化时。  
> **谁改**：默认由改该流程的人先改。  
> **是否 review**：强烈建议；最好由熟悉业务的人核对与代码/接口/表一致。

---

## 1. 用户注册（企业认证）流程

- **触发条件**：用户在小程序/端侧填写企业信息并提交。
- **入口接口**：`POST /api/auth/register`。
- **涉及模块**：server（校验、落库）、miniapp（表单与提交）。
- **涉及表**：`enterprise_applications`（及可能的 users 关联）。
- **状态变化**：申请创建为 `pending`，等待运营审核。
- **异常处理**：参数错误返回 4xx；重复提交策略见 openspec/specs/auth-register。
- **风险点**：一用户一有效申请策略、营业执照与敏感信息存储。

---

## 2. 登录流程

- **触发条件**：用户输入手机号与验证码并提交。
- **入口接口**：`GET /api/auth/captcha` 获取验证码，`POST /api/auth/login` 登录。
- **涉及模块**：server（验证码生成/校验、JWT 签发）、miniapp（登录页与鉴权头）。
- **涉及表**：users（若为首次登录可能创建）、验证码内存/存储（见实现）。
- **状态变化**：登录成功后返回 JWT；前端持久化 token，后续请求带 `Authorization: Bearer <token>`。
- **异常处理**：验证码错误/过期、手机号格式错误返回明确错误信息。
- **风险点**：生产环境 JWT_SECRET、验证码实现需替换为安全方案。

---

## 3. 企业审核流程

- **触发条件**：审核员或超级管理员在后台打开「企业管理」模块的申请视图，对某条申请执行通过/驳回。
- **入口接口**：`POST /api/admin/login` 登录后获取带 `adminId/username/role` 的管理员 JWT；申请列表/详情使用 `GET /api/admin/reviews`、`GET /api/admin/reviews/:id`，审核动作使用 `POST .../approve` / `.../reject`。
- **涉及模块**：server、admin-frontend。
- **涉及表**：`enterprise_applications`（status、audit_at、audited_by、reject_reason 等）。
- **状态变化**：`pending` → `approved` 或 `rejected`；用户端通过 `GET /api/auth/audit-status` 查询。
- **异常处理**：未登录管理端、无权限、重复审核等返回相应错误；高风险动作（档案编辑、测试数据删除）仅允许 `super_admin`。
- **风险点**：审核后不可随意改状态；驳回原因与审核人需留痕。

## 4. 采集任务流程（控制面）

- **触发条件**：scheduler 定时触发或运营在后台发起「采集控制」。
- **入口**：scheduler 调用 `scripts/scheduler_submit.py` 向 `POST /api/admin/crawl/runs` 提交；或前端通过同一控制面提交。
- **涉及模块**：server（dispatcher 认领、子进程执行）、crawler（site1/site2.tasks.*）、scheduler。
- **涉及表**：crawl_runs（及可能的队列表，见实现）。
- **状态变化**：run 创建为 queued → 被 api 进程认领 → 执行中 → success/failure；运行记录可查 `GET /api/admin/crawl/runs`。
- **异常处理**：子进程超时、崩溃、站点不可用；日志落 `logs/admin-crawl/`，run 状态更新为失败。
- **风险点**：单实例 API 下同一时间仅一个 run 执行；高危动作（如 formal backfill）禁止从后台发起。

---

## 5. 公告数据流

- **触发条件**：爬虫任务执行（incremental / backfill / recovery）。
- **入口**：`python -m crawler.site1.tasks.incremental` 等。
- **涉及模块**：crawler、storage、server（只读）。
- **涉及表**：notices（upsert 以 (site, id) 为唯一键）；字典表由爬虫或脚本维护。
- **状态变化**：新公告插入，已有公告更新 last_seen_at 及可更新字段；无单独“状态”字段时以存在与否表示。
- **异常处理**：网络/代理失败重试；对账任务发现源与库不一致时可打报告或触发补偿。
- **风险点**：site2 依赖代理与验证码；大批量 backfill 需控制窗口与频率。

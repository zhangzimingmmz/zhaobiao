# miniapp-data 规格

## ADDED Requirements

### Requirement: 接口契约与接口文档一致

小程序发起的列表、详情、登录、注册、收藏等请求的 URL、方法、请求体与响应结构 MUST 遵循仓库内 `接口文档.md` 的约定。不新增或修改后端接口；与 Web 端共用同一套 API 与数据契约。

#### Scenario: 列表请求符合文档约定

- **WHEN** 小程序请求招投标或信息展示列表
- **THEN** 使用的接口路径、参数（如分页、筛选条件）与 `接口文档.md` 中对应接口描述一致

#### Scenario: 详情请求符合文档约定

- **WHEN** 小程序请求招投标详情或信息展示详情
- **THEN** 使用的接口路径与参数（如 id、planId 等）与 `接口文档.md` 中对应接口描述一致

### Requirement: 登录与注册流程可调通

小程序 MUST 提供登录与注册（企业认证）的请求封装，请求格式与 `接口文档.md` 中相关约定一致（若文档已定义）。验证码获取、提交登录、提交注册等 MUST 能正确调用后端（或 Mock），并在前端展示成功/失败与跳转。

#### Scenario: 登录接口调用与结果处理

- **WHEN** 用户输入手机号与验证码并点击登录
- **THEN** 小程序按接口文档发起登录请求，并根据响应结果跳转（成功进入首页或我的）或展示错误信息

#### Scenario: 注册提交与审核状态跳转

- **WHEN** 用户完成企业信息填写并提交注册
- **THEN** 小程序按接口文档提交注册请求，并支持跳转到审核状态页；审核状态页能展示审核中/通过/未通过三种状态（数据来源按接口文档或 Mock）

### Requirement: 收藏状态与详情原文可请求

招投标详情页的收藏状态 MUST 能与后端同步（若接口文档中有收藏接口）；详情页「查看原文」与信息展示详情页「公众号原文」MUST 使用接口或数据中提供的链接进行跳转。

#### Scenario: 详情页收藏可切换并同步

- **WHEN** 用户在招投标详情页点击收藏按钮
- **THEN** 本地状态更新，且若后端提供收藏接口则发起对应请求以保持与服务端一致

#### Scenario: 原文链接可跳转

- **WHEN** 用户在详情页点击「查看原文」或「公众号原文」
- **THEN** 使用接口返回或数据中的原文 URL 进行跳转（小程序内 web-view 或复制链接等，按平台能力与产品约定实现）

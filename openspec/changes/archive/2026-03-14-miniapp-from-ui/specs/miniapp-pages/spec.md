# miniapp-pages 规格

## ADDED Requirements

### Requirement: 七类页面与 ui 对应

小程序 MUST 实现与 `ui/PAGES_OVERVIEW.md` 对应的 7 类页面：首页（Home）、招投标详情（Detail）、信息展示详情（InfoDetail）、登录（Login）、注册/企业认证（Register）、审核状态（AuditStatus）、我的（Profile）。每类页面的功能模块与入口 MUST 与 PAGES_OVERVIEW 中描述一致。

#### Scenario: 首页包含一级/二级菜单与筛选

- **WHEN** 用户进入首页
- **THEN** 可见顶部导航、一级菜单（工程建设/政府采购/信息展示）、根据一级菜单变化的二级菜单、筛选区与列表区、底部导航（首页/我的）

#### Scenario: 详情与登录注册审核流程可访问

- **WHEN** 用户通过导航或链接进入任意上述页面
- **THEN** 对应页面正确渲染，且具备 PAGES_OVERVIEW 中列出的主要模块（如详情页含返回、标题、收藏、原文；登录页含手机号、验证码、登录按钮、注册入口等）

### Requirement: 页面路由与导航关系正确

小程序路由与页面栈 MUST 支持：登录 → 注册 → 审核状态 → 首页 ↔ 我的；首页列表项 → 招投标详情或信息展示详情；我的等入口进入对应子页。底部 Tab MUST 在首页与我的之间切换。

#### Scenario: 从首页进入详情并返回

- **WHEN** 用户在首页点击某条招投标或信息展示列表项
- **THEN** 进入对应详情页（招投标详情或信息展示详情），且可通过返回回到首页或上一级

#### Scenario: 底部导航切换首页与我的

- **WHEN** 用户在首页或我的页面点击底部导航的「首页」或「我的」
- **THEN** 切换到对应 Tab 页面，且状态符合 PAGES_OVERVIEW 描述

### Requirement: 菜单与筛选联动与 ui 一致

首页一级菜单（工程建设/政府采购/信息展示）与二级菜单联动、筛选区随当前菜单组合变化的行为 MUST 与 PAGES_OVERVIEW 中「菜单联动关系」及「筛选区」描述一致（含工程建设-工程建设、工程建设-政府采购、政府采购-采购意向公开、政府采购-采购公告、信息展示仅搜索等）。

#### Scenario: 切换一级菜单更新二级菜单与筛选

- **WHEN** 用户切换一级菜单（如从「工程建设」改为「政府采购」）
- **THEN** 二级菜单与筛选区更新为 PAGES_OVERVIEW 中该组合对应的选项

#### Scenario: 信息展示无二级菜单

- **WHEN** 用户选择「信息展示」
- **THEN** 仅显示搜索相关筛选，无二级菜单

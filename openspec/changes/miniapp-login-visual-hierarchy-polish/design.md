## Context

当前登录页已经有品牌区、登录区和游客预览区，但视觉上仍然存在三个问题：一是登录区虽然处于中段，却没有成为唯一强主视觉；二是输入框采用浅灰块样式，容易让用户误判为弱态或不可编辑；三是注册入口、协议区和预览区的权重没有充分退后，导致整页层次偏平。现有实现已经完成数据和页面结构切分，因此这次不需要改业务逻辑，只需要收敛视觉语言和布局节奏。

## Goals / Non-Goals

**Goals:**
- 让登录表单成为首屏最强视觉焦点，品牌区和预览区只承担辅助角色。
- 将输入区调整为独立字段标题 + 白底细边框输入框的表单风格，消除“灰块弱态”观感。
- 强化主登录按钮的重量感，并系统性弱化注册入口、协议文案、分割元素和外层卡片存在感。
- 保持游客预览作为登录页底部辅助模块，但让它继续贴近正式消息卡片的阅读顺序。

**Non-Goals:**
- 不调整登录鉴权、注册流程、审核分流或游客拦截逻辑。
- 不新增新的登录字段、验证码、找回密码或企业认证动作。
- 不修改后端 API、数据源、轮播取数规则或整体页面路由。

## Decisions

### 1. 保留三段骨架，但将视觉重心转移到登录区
- 继续沿用品牌区 / 登录区 / 预览区的纵向骨架，避免再次重构页面结构。
- 登录区在版式上保持最大占比，并通过更强的输入框与按钮样式建立主次，而不是只靠容器高度解决问题。
- 品牌区仅保留 logo / 图形与 `金堂招讯通` 标题，不再在品牌卡内堆叠说明文字。

**Why this over alternatives**
- 比完全取消品牌区更稳，仍能保留小程序品牌识别。
- 比继续放大品牌卡更好，因为页面主任务仍然是登录。

### 2. 输入框改为“白底 + 细边框 + 独立字段标题”
- 登录名和登录密码字段使用独立标题文本放在输入框上方。
- 输入框样式统一切到白底、浅边框、明确可编辑态，避免继续使用浅灰块式背景。
- 焦点态沿用品牌蓝强化边框或视觉反馈，形成明确的可输入感。

**Why this over alternatives**
- 比使用 Taro UI 自带横向 `title` 更适合当前移动端表单版式，不会压缩输入区宽度。
- 比只保留 placeholder 更清楚，且能自然填充表单内部空白。

### 3. 主按钮增强，次级信息系统性降级
- `登录` 保持为唯一强蓝主按钮，可通过尺寸、阴影或对比进一步增强重量感。
- `去注册` 保留为文本入口，但从主色高亮中降为次级链接或弱化色。
- 协议文案移除不必要的分割线或显著结构线，避免打断视线。
- 外层登录卡继续保留，但弱化边框、阴影和“老后台大面板”观感。

**Why this over alternatives**
- 比删除注册入口更符合当前业务需要。
- 比继续保留强分割线更能让用户视线自然集中在主按钮。

### 4. 预览区维持辅助身份，不与表单争夺主视觉
- “最新招采预览”继续位于底部，并保持单条轮播。
- 单条预览继续参考正式消息卡片顺序：标题、分类标签、来源、发布时间。
- 预览卡保持轻量和低权重，不增加收藏、摘要、主体等字段，避免重新压过登录区。

**Why this over alternatives**
- 比完全移除预览区更不利于审核和内容展示。
- 比恢复多条大卡轮播更容易破坏登录页的主次关系。

## Risks / Trade-offs

- [风险] 输入框边框增强后，若品牌区和预览区仍然过亮，页面可能出现“上中下都在抢”的问题。  
  → Mitigation: 同步弱化品牌卡描边和预览区容器存在感，确保只有登录区形成强对比。

- [风险] 弱化注册入口过头，首次使用用户可能忽略注册路径。  
  → Mitigation: 保留文本入口和清晰文案，但不再赋予与主按钮相同的颜色权重。

- [风险] 去掉协议分割线后，底部信息可能显得过松散。  
  → Mitigation: 通过较小字号、统一灰度和收紧 spacing 维持层次，而不是依赖显性结构线。

- [风险] 预览卡继续留在页面底部，会在部分长屏设备上显得较空。  
  → Mitigation: 让登录区自身内容更完整，用字段标题、表单节奏和更明确的输入态填满主模块，而不是靠增加次级信息填空。
